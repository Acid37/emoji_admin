# emoji_admin

表情包管理后台插件。

## 背景

为了使 Bot 发送表情包更加**应景**，特此制作了这样一款 Web 管理后台。

在 LLM 仪表盘统计和 Memory 前端设计的启发下，有了这一款依赖 `emoji_sender` 运行的可视化管理工具——相对轮椅，可视化。

支持修改表情包的 Tag 等数据库操作，所有修改都会同步重建向量记录。

## 功能

- 图片预览
- 表情包列表检索
- Tag 编辑
- 删除表情包
- 仅看文件异常的筛选
- 根据同步时间提示下一次刷新时机

## 数据来源

本插件不修改 `emoji_sender` 的发送逻辑，只复用同一套数据：

- 图片目录：`data/emoji_sender/memes`
- 向量库：`data/emoji_sender/vector_db`
- 集合名：`emoji_sender`

## 访问方式

启动插件后访问：`http://<你的服务器地址>:<端口>/emoji-admin/`

端口号在 `config/core.toml` 中的 `http_router_port` 项查看，默认通常是 `8000`。

## 目录结构

- `plugin.py`：插件入口
- `service.py`：数据读取、更新、删除逻辑
- `router/emoji_admin_router.py`：HTTP 路由
- `router/emoji_admin.html`：前端页面

## 说明

这是一个独立外挂插件，用来和远程仓库内置的 `emoji_sender` 解耦。
更新后台界面或管理逻辑时，不需要改原始发送插件。

## 同步说明

`emoji_sender` 的同步间隔会从当前部署的配置文件 `config/plugins/emoji_sender/config.toml` 动态读取，后台页面会直接显示这份真实配置，而不是写死一个固定数字。

页面还会基于**本机本地时间**和最近一次同步记录，动态计算下一次可刷新时间点；如果当前还没看到更新，可以直接看页面提示在什么时候刷新。

如果页面中出现“文件异常”，它表示数据库里有这条表情包记录，但本地图片暂时没找到；这时可以切换“仅看文件异常”快速筛选，也可以先等到页面提示的下次刷新时间再查看。

插件加载后还会先触发一次立即入库，所以如果你刚启动系统，通常会在启动后很快看到第一轮更新。

如果某些表情包暂时没显示，通常建议先刷新页面，再等待页面给出的下一次刷新时间；具体间隔以当前部署读取到的配置为准。
